# m-index-hub

## 项目简介

`m-index-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。项目本身不针对任何特定域名或网站，仅作为页面资源的集合与索引，方便维护和访问。

## 仓库结构

```
m-index-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
├── index.html      # 页面索引文件（可选）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件为一个独立页面，命名清晰，便于识别内容。
- **assets/**：存放页面共用的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选的入口页面，用于列出所有归档页面的链接或说明。

## 页面归档说明

本仓库收录的页面均为独立的 HTML 文档，内容可能涉及工具、信息展示、交互演示等多种类型。每个页面保持自包含性，尽量减少外部依赖。

归档遵循以下原则：
- 页面文件命名使用英文小写字母、数字与短横线，如 `example-page.html`。
- 如有需要，可在页面头部添加简要注释说明用途或版本。
- 不存储任何与具体网站或域名相关的敏感信息。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并确保引用资源路径正确。
- 修改或删除页面时，建议同步更新 `index.html`（若存在）以保持索引准确。
- 公共资源如样式或脚本，建议统一放在 `assets/` 下，避免重复。
- 本仓库不鼓励存放动态后端代码或数据库相关内容，保持纯前端静态页面结构。
